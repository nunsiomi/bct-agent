01 — Overview

Two Tasks. One Shared Brain.

The BCT challenge asks teams to build LLM-based agents that understand how people behave, what they want, and what they'll choose next. We solve this with a shared persona builder that feeds two task-specific agent pipelines — both containerised and independently deployable.

Task A

User Modeling

Takes a user persona + product details as input. Outputs a simulated star rating and written review that captures the user's tone, rating behaviour, and cultural context — including Nigerian English and Pidgin markers.

Task B

Recommendation

Takes a user persona + domain as input. Routes through domain validation, retrieval, and LLM reasoning to produce a ranked, explained recommendation list — handling cold-start, cross-domain, and niche queries.

Key design principle: Datasets are consumed at development time only. The runtime app accepts free-text persona descriptions with no raw data access — behavioral understanding is baked into the agent's prompt architecture.

02 — Framework Selection

Why LangGraph over CrewAI and OpenClaw

We evaluated three agentic frameworks before committing. The decision came down to control, observability, and how well each maps to a scoreable solution paper.

Criterion OpenClaw CrewAI LangGraph CHOSEN
Maturity ✗ Relatively early-stage, limited community ✓ Active community, good docs ✓✓ Backed by LangChain team, production-grade
State control ✗ Limited explicit state management ⚠️ Implicit state via role-based agents ✓✓ Explicit typed state schema — every node reads/writes a defined state object
Conditional routing ✗ Hard to implement custom edges ⚠️ Possible but not first-class ✓✓ Conditional edges are a core primitive — perfect for our fallback logic
Debuggability ✗ Black-box execution ⚠️ Some visibility ✓✓ Full graph trace in LangSmith — every node input/output inspectable
Solution paper fit ✗ Hard to diagram or explain ⚠️ Role metaphor can be described ✓✓ Graph nodes + edges map directly to an architecture diagram — judges can read both
Multi-graph support ✗ No native concept ⚠️ Separate crews required ✓✓ Two independent graphs sharing a common node — exactly our architecture
Nigerian context injection ⚠️ Prompt-only ⚠️ Prompt-only ✓✓ Dedicated context node with typed state injection at every downstream node

The core reason

LangGraph treats an agent as a directed graph with typed state. Every node is a function that reads from and writes to a shared state object. Every edge is explicit — including conditional edges that route based on state values. This means our domain validation fallback, our Nigerian context injection, and our cold-start routing are all first-class, inspectable, testable code — not prompt hacks buried in a role description.

For a competition where the solution paper is read first, this matters enormously. A LangGraph architecture can be drawn as a diagram and explained node by node. CrewAI's role-based metaphor is harder to map to a precise architecture diagram, and OpenClaw lacks the community and tooling to make this credible under a 6-day deadline.

05 — Data Workflow

How All Three Datasets Are Consumed

Each dataset contributes distinct behavioral signals that feed into different parts of the persona schema. Elite Yelp users are prioritised for sampling based on findings from the analysis reports — they have 5–7× more reviews and more consistent rating patterns.

Yelp

Behavioral richness

Primary source. Extract avg_stars, review_count, yelping_since, elite status, category affinity, price range sensitivity. Filter for elite users first — 5–7× richer profiles.

Amazon

Product intent

Secondary source. Extract purchase-intent signals, rating bias per product category, verified vs. unverified review patterns. Most useful for Task B product domain recommendations.

Goodreads

Genre taste

Tertiary source. Extract genre affinity from shelf behaviour, reading velocity, review length distribution. Most relevant for Task B when domain is Books.

Calibration insight from uploaded reports: The zzhang83 analysis found that 56.5% of fake reviews are 5-star while true reviews cluster at 3.5–4 stars. Our quality checker node caps confidence on 5-star outputs accordingly. The hemachandarn report confirmed the elite user sampling strategy — dropping the friends column and filtering elite_ever = true reduces data size from 720MB to 10MB per file while retaining the most behaviorally rich users.

UI Design Decisions

No dataset selector

Why we removed it

Datasets are consumed at dev time only. The UI accepts free-text persona descriptions — the behavioral understanding is already baked into the agent's prompt architecture. A dataset dropdown would imply runtime data access that doesn't exist.

Graceful fallback

Unrecognised domain handling

When the domain validator can't match the user's input, the UI surfaces two choices: retype for clarity, or proceed with general lifestyle recommendations. This gives the user agency rather than silently failing — and demonstrates defensive agent design.

07 — LLM Selection

Powered by Claude API

All LLM calls across both tasks — persona fingerprinting, review generation, reasoning, and ranking — are made through the Anthropic Claude API using claude-sonnet-4-20250514. Claude is chosen for its nuanced instruction-following, long-context reasoning, and strong performance on text generation tasks requiring cultural and tonal specificity.

Model

claude-sonnet-4-20250514

Optimal balance of quality and speed for this use case. Strong at persona-conditioned generation, structured JSON output, and multi-turn reasoning — all required across our LangGraph nodes.

Context window

200K tokens

Allows the full persona fingerprint, behavioral history summary, domain context, and Nigerian localization instructions to be injected in a single prompt without truncation.

Nigerian context

Instruction-following fidelity

Claude reliably follows nuanced style instructions — switching register to Nigerian Pidgin, anchoring reviews in local price references (₦), and producing culturally grounded recommendation reasoning.

Structured output

JSON-mode generation

Both task outputs are requested as structured JSON — review text + star rating for Task A, ranked recommendation list with match scores and reasoning for Task B. Claude returns clean, parseable responses without fences.

How Claude sits inside LangGraph

Each LangGraph node that requires LLM reasoning calls the Claude API directly via the Anthropic Python SDK. The node receives the current AgentState, constructs a targeted prompt, calls Claude, parses the response, and writes the result back to state. No LangChain LLM wrappers are used — direct API calls keep the implementation transparent and easy to debug in LangSmith.

# Example: persona builder node calling Claude
import anthropic

client = anthropic.Anthropic()

def persona_builder_node(state: AgentState) -> AgentState:
    prompt = f"""
    You are a behavioral analyst. Extract a structured persona fingerprint
    from this description. Include: rating_bias (1-5 float), tone (formal/casual/pidgin),
    price_sensitivity (low/medium/high), category_affinity (list),
    nigerian_markers (list of cultural signals).

    Persona: {state['persona_text']}

    Respond with JSON only, no preamble.
    """
    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=1000,
        messages=[{"role": "user", "content": prompt}]
    )
    state['fingerprint'] = json.loads(response.content[0].text)
    return state
08 — Evaluation Metrics

How the Submission Is Scored

The competition uses a 100-point rubric. Task A metrics are qualitative and automated; Task B metrics are heavily quantitative with human evaluation for contextual relevance. Understanding this scoring shapes every design decision.

Task A — User Modeling

Review Text Quality

ROUGE / BERTScore

ROUGE-L measures longest common subsequence overlap between generated and reference reviews — rewards fluency and structure. BERTScore F1 uses contextual embeddings to measure semantic similarity — more forgiving of paraphrasing. Target: ROUGE-L > 0.55, BERTScore > 0.78.

Rating Accuracy

RMSE

Root Mean Square Error between predicted star rating and ground truth. Lower is better. Our quality checker caps 5-star confidence based on real Yelp distribution data — most genuine reviews cluster at 3.5–4 stars — which keeps RMSE in range.

Human Evaluation

Behavioural Fidelity

Judges assess whether the generated review sounds like it came from the described persona — tone match, rating consistency, cultural authenticity. This is where Nigerian contextualization earns bonus marks. Pidgin usage, ₦ price references, and local food vocabulary all contribute.

Task B — Recommendation (100 pts breakdown)

Criterion Metric Points Our approach
Ranking Quality NDCG@10 / Hit Rate 30 Embedding retrieval + Claude re-ranking. Reasoning node justifies each position, improving NDCG over pure similarity search.
Cold-Start & Cross-Domain Held-out user eval 25 Domain-agnostic persona fingerprint handles cold-start. Open "Other" domain field with niche narrowing handles cross-domain explicitly.
Contextual Relevance Human evaluation 20 Claude's reasoning node explains each recommendation in persona-specific terms. Nigerian context layer surfaces culturally resonant picks.
Solution Paper Judge review (4–8 pages) 15 This documentation is the paper's technical backbone. Architecture diagrams, node-edge explanations, dataset methodology, and ablation notes all drawn from here.
Code Reproducibility Judge attempts to run it 10 Docker Compose spins up both containers. Clear README, modular LangGraph node structure, well-commented conditional edges and fallback logic.

Key scoring insight: The brief explicitly states "the solution paper is what we read first" and that "additional marks" are awarded for Nigerian contextualisation. Both of these are baked into our architecture from day one — not afterthoughts. The Nigerian context node injects localization at the LangGraph state level, ensuring every downstream Claude call inherits it automatically.
